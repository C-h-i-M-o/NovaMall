# NovaMall 总体设计规格

## 目标

从零构建一个名为 NovaMall（星选）的多商户网上购物系统，完成课程规定业务，并以可复现实验展示 9 项高阶数据库技术。

## 已确认决策

- 技术栈：Express + React + MySQL，前后端均使用 TypeScript。
- 角色：会员、店主、系统管理员。
- 商业模式：多商户；一个店主经营一家店铺。
- 入驻：会员申请，管理员审核后创建店铺并授予店主角色。
- 结算：购物车可跨店，一次结算自动拆分子订单。
- 支付：模拟支付，不接第三方渠道。
- 售后：待发货子订单整单退款，由店主审核。
- 热销榜：全平台累计有效销量 Top 10，排除取消和退款。
- 图片：本地上传，数据库保存访问路径。
- 登录：MySQL Session + HttpOnly Cookie。
- 加密：手机号 AES 加密，地址明文，密码不可逆哈希。
- 部署：Docker Compose。
- 数据访问：mysql2 显式 SQL，不使用 ORM。

## 功能范围

会员完成注册、登录、商品浏览和查找、购物车、幂等跨店结算、支付、订单查询、收货和退款申请；店主完成店铺、商品、库存、订单发货和退款审核；管理员完成账号、分类、开店审核和审计查询。

## 架构

采用一个 React 前端、一个 Express API 和一个 MySQL 数据库的模块化单体。后端固定使用 `Route → Controller → Service → Repository` 调用方向。

## 数据模型

核心实体包括用户、角色、地址、开店申请、店铺、分类、商品、购物车、总订单、店铺子订单、订单明细、支付、退款、价格历史、审计和 Session。详细字段、关系、范式和约束见 `docs/database/`。

## 高阶数据库技术

实现存储过程、触发器、视图、索引优化、事务并发、窗口函数、审计日志、全文检索和 AES 数据加密。每项必须提供 SQL、测试数据、执行步骤、期望结果和真实证据。

## 安全与错误处理

所有输入通过 Zod 校验；SQL 使用参数化查询；受保护资源验证角色和归属；Cookie 使用 HttpOnly 和 SameSite；生产环境使用 Secure。错误响应包含稳定错误码和请求编号，不泄露内部信息。

## 测试

使用单元测试、React 组件测试、API/MySQL 集成测试、并发测试和关键 E2E 测试。每阶段必须通过类型检查、Lint、测试、构建和 Docker 健康检查。

## 交付策略

按 `docs/development-plan.md` 分阶段实施。每阶段先完成文档与验收定义，用户确认后再实现，并在验证通过后进入下一阶段。

## 非目标

不实现 SKU、优惠券、真实支付、真实物流、复杂退货、多店员工、微服务、Redis、消息队列和 Elasticsearch。
