# NovaMall 分阶段开发计划

## 1. 执行原则

每个阶段都是可验证的最小闭环，严格执行：

```text
阶段文档 → 用户确认 → 失败测试 → 最小实现 → 全量验证 → 阶段验收
```

开始阶段前，在 `docs/phases/` 创建阶段规格，至少包含需求、非目标、API、数据变更、实现方案、测试和验收命令。阶段规格未经确认，不写该阶段代码。

## 2. 阶段 0：设计与文档基线

### 范围

- README、AGENTS；
- 需求、架构、ER、范式和完整性；
- 高阶数据库技术与实验计划；
- API、安全、UI、测试、部署和课程报告文档；
- 分阶段开发计划。

### 验收

- 文档无 TBD/TODO 和相互冲突；
- Mermaid 图可解析；
- 表名、状态、角色和 API 命名一致；
- 用户确认文档后才能进入阶段 1。

## 3. 阶段 1：基础设施与认证闭环

### 文档先行

创建 `docs/phases/01-foundation-auth.md`，确认 workspace、环境变量、Session、用户表、角色表和认证 API。

### 实现

- pnpm workspace、React、Express、共享类型和 Docker Compose；
- MySQL 迁移机制和基础健康检查；
- users、roles、user_roles、sessions；
- 注册、登录、退出、当前会话；
- scrypt 密码哈希、手机号 AES、CSRF、统一错误和请求编号；
- 三角色路由骨架。

### 最小闭环

会员可注册、登录、查看加密后的个人资料并退出；管理员和店主测试账号能进入各自空后台；越权请求被拒绝。

### 验收

- 数据库中无明文密码和手机号；
- 登录轮换 Session，退出后失效；
- 三角色权限集成测试通过；
- Docker 从空卷启动。

## 4. 阶段 2：商户入驻闭环

### 文档先行

创建 `docs/phases/02-merchant-onboarding.md`，确认申请状态机、审核事务和店铺字段。

### 实现

- merchant_applications、shops；
- 会员提交、查看和拒绝后重提；
- 管理员申请列表、批准和拒绝；
- 批准事务创建店铺并授予 OWNER 角色；
- 审核审计触发器；
- 开店申请和管理页面。

### 最小闭环

普通会员提交申请，管理员批准后，该会员无需重新注册即可成为单店店主并进入店主后台。

### 验收

- 一人一店和一个当前申请约束有效；
- 并发重复审核只有一个成功；
- 事务失败不产生孤立店铺或角色；
- 审计日志正确。

## 5. 阶段 3：商品目录闭环

### 文档先行

创建 `docs/phases/03-catalog.md`，确认分类、商品、图片、搜索、索引和页面状态。

### 实现

- categories、products、product_price_history；
- 管理员分类管理；
- 店主商品 CRUD、库存、上下架、归档和图片上传；
- 公开商品列表、分类筛选、详情和中文全文检索；
- 商品价格历史和关键审计触发器；
- 商品查询复合索引与 FULLTEXT ngram 索引。

### 最小闭环

管理员创建分类，店主发布带图片商品，游客或会员按分类和中文关键词找到商品并查看详情。

### 验收

- 店主不能操作其他店铺商品；
- 下架、停用分类和停用店铺不出现在公开列表；
- 非图片和超限文件被拒绝；
- 价格变化自动生成历史；
- 保存索引和全文检索初步实验结果。

## 6. 阶段 4：购物车与跨店结算闭环

### 文档先行

创建 `docs/phases/04-cart-checkout.md`，确认订单字段、存储过程接口、锁顺序、错误码和幂等策略。

### 实现

- addresses、cart_items、master_orders、shop_orders、order_items、payments；
- 地址和购物车 CRUD；
- `sp_checkout_cart`；
- 跨店拆单、订单快照、库存扣减和购物车清理；
- 结算页面和待支付订单详情。

### 最小闭环

会员将两个店铺商品加入购物车，一次结算后得到一个总订单和两个子订单，库存正确扣减。

### 验收

- 存储过程输入/输出测试通过；
- 库存不足时全事务回滚；
- 同一 checkoutToken 重试不重复下单；
- 两请求争抢最后库存时仅一个成功；
- 金额三层一致，历史快照不受商品修改影响。

## 7. 阶段 5：支付、履约与排行闭环

### 文档先行

创建 `docs/phases/05-payment-fulfillment.md`，确认总/子订单状态机、支付幂等、发货和确认规则。

### 实现

- 模拟支付事务；
- 取消待支付订单并恢复库存；
- 店主子订单列表和发货；
- 会员订单查询和确认收货；
- 订单详情视图、销售汇总视图和有效销量视图；
- 窗口函数 Top 10；
- 订单状态审计。

### 最小闭环

会员支付跨店订单，两位店主独立发货，会员分别确认收货，总订单正确聚合完成。

### 验收

- 重复支付不重复记账；
- 跨店发货和跨会员确认被拒绝；
- 取消订单恢复库存；
- Top 10 排除取消订单且并列排序稳定；
- 视图和基础表结果一致。

## 8. 阶段 6：简单退款闭环

### 文档先行

创建 `docs/phases/06-refunds.md`，确认退款状态机、库存恢复和总订单聚合规则。

### 实现

- refund_requests；
- 会员待发货子订单整单退款申请；
- 店主批准或拒绝；
- 批准事务恢复库存并更新聚合状态；
- 退款页面和审计。

### 最小闭环

会员申请退款，店主分别执行批准和拒绝；批准分支恢复库存，拒绝分支恢复待发货状态。

### 验收

- 已发货、已完成和跨会员申请被拒绝；
- 跨店审核被拒绝；
- 每个子订单只产生一个退款申请；
- 退款事务失败全部回滚；
- Top 10 排除已退款销量。

## 9. 阶段 7：数据库技术与管理闭环

### 文档先行

创建 `docs/phases/07-database-evidence-admin.md`，确认性能数据规模、实验命令和管理员演示内容。

### 实现

- 管理员用户、订单只读和审计页面；
- 店主与平台经营汇总；
- 数据库技术只读演示页；
- 性能数据生成器；
- 9 项技术的自动测试和证据文档。

### 最小闭环

管理员能从界面查看平台数据和审计，并通过可复制命令复现全部数据库实验。

### 验收

- 完成 `docs/evidence/database/` 全部证据；
- 索引和全文检索报告包含真实数据规模、执行计划和计时；
- 审计不含敏感字段；
- 数据库演示接口不能执行任意 SQL。

## 10. 阶段 8：最终交付闭环

### 文档先行

创建 `docs/phases/08-release.md`，确认演示数据、部署步骤、报告内容和最终验收脚本。

### 实现

- 完整 Docker Compose、健康检查和持久化卷；
- `.env.example` 和演示数据；
- README 运行、测试和故障排查；
- 课程报告、ER 图和实验结果；
- 完整关键 E2E。

### 最小闭环

在空环境一键启动，按最终演示脚本完成三角色、两店铺、跨店订单、退款和 9 项数据库技术展示。

### 验收

- 全量类型检查、Lint、测试、构建和 E2E 通过；
- 空卷启动和容器重启测试通过；
- 文档与实现一致；
- 无 Secret、真实数据和临时文件进入 Git；
- 课程报告中的数字均可由证据复现。

## 11. 阶段变更规则

- 新需求先判断属于当前阶段还是后续阶段；
- 不为后续需求提前抽象；
- 若改变已确认表结构或接口，先更新阶段文档并重新确认；
- 阶段验收失败时只修复该阶段范围，不并行推进下一阶段。
